
# Tutorial: Github Content Manager Agent

This Agent is designed to help you manage your GitHub repository content. The capabilities include retrieving the content of a repository, including specific files like `README.md`. It can also create and update file content. The aim is to help you navigate and handle your GitHub repositories with ease.

<Youtube videoId="HCzyC1vAd7Q" title="SmythOS - Github Content Manager Agent" />

## GitHub Access Tokens

1. Navigate to ![https://github.com](https://github.com).
2. In the upper-right corner of the page, click your profile photo.

<Image src="/docs/img/github_content_manager_agent/Image 1.png" alt="alt text"/>

3. Then, click `Settings`.

<Image src="/docs/img/github_content_manager_agent/Image 2.png" alt="alt text"/>

4. In the left sidebar, scroll down then click `Developer settings`.

<Image src="/docs/img/github_content_manager_agent/Image 3.png" alt="alt text"/>

5. In the left sidebar, click `Personal tokens` then click `Generate New Token (classic)`.

<Image src="/docs/img/github_content_manager_agent/Image 4.png" alt="alt text"/>

6. Add a note to your `Token`, set the expiration, and then checkmark the scopes.

<Image src="/docs/img/github_content_manager_agent/Image 5.png" alt="alt text"/>

7. Then, click the `Generate Token` button.

<Image src="/docs/img/github_content_manager_agent/Image 6.png" alt="alt text"/>

8. Make sure to copy your personal access token because you won’t be able to see it again.
   1. **Example:** `ghp_wtOAM5BSYSCgnTN5elXEUAerngae4P3pqFmc`

<Image src="/docs/img/github_content_manager_agent/Image 7.png" alt="alt text"/>

## GitHub Repository Owner & Name

> You can skip this step if you already have your Repositories.

1. In the upper-right corner of the `GitHub` page, click your profile photo.

<Image src="/docs/img/github_content_manager_agent/Image 8.png" alt="alt text"/>

2. Then, click `Settings`.

<Image src="/docs/img/github_content_manager_agent/Image 9.png" alt="alt text"/>

3. In the left sidebar, go to `Code, Planning, and Automation` then click `Repositories`.

<Image src="/docs/img/github_content_manager_agent/Image 10.png" alt="alt text"/>

4. Click `create one`.

<Image src="/docs/img/github_content_manager_agent/Image 11.png" alt="alt text"/>

5. The owner’s name will be automatically created, you only need to input your repository name.
   1. **Example:**
      1. **Repository Owner:** `janicesmyth`
      2. **Repository Name:** `SmythOS-GitHub-Manager`

<Image src="/docs/img/github_content_manager_agent/Image 12.png" alt="alt text"/>

6. Then, click the  `Create Repository` button.

<Image src="/docs/img/github_content_manager_agent/Image 13.png" alt="alt text"/>
## Agent Setup

1. In SmythOS, navigate to the Templates tab.

<Image src="/docs/img/github_content_manager_agent/Image 14.png" alt="alt text"/>

2. You can scroll down to find the `GitHub Content Manager` or simply search for it in the Search tab, then click the `Remix` button.

<Image src="/docs/img/github_content_manager_agent/Image 15.png" alt="alt text"/>

3. Allow the template to initialize and configure.

## GitHub Integrations Setup

1. In `SmythOS`, look for `GitHub` integration components. This agent utilized five `GitHub` component integrations. Click the gear icon to open the settings.

<Image src="/docs/img/github_content_manager_agent/Image 16.png" alt="alt text"/>

2. Click the key icon and select your `GitHub Access Token` stored in your vault. Remember to click the check mark to save your configuration.
3. >**Note:** Click [here](/docs/agent-studio/key-concepts/vault?id=keys) to learn more about adding your API keys to the vault.

<Image src="/docs/img/github_content_manager_agent/Image 17.png" alt="alt text"/>

4. You’re all set!
